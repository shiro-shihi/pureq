# Changelog

All notable changes to @pureq/db are documented in this file.

## [1.2.0] - The "Fortress Edition" Release

### Major Performance Optimizations

- **v1.2 Bitwise Engine**: Replaced `DataView` with raw bitwise operations (`<<`, `|`, `&`) for integer parsing. This eliminates alignment checks and allows the V8 JIT compiler to generate near-native machine code, resulting in a **1.9x to 3.8x speedup** in row decoding.
- **Ultra-Fast ASCII Decoder**: Implemented a JS-native fast-path for ASCII string decoding that bypasses the C++/JS boundary crossing of the standard `TextDecoder`.
- **Zero-Copy Buffer Management**: Optimized `BufferReader` to return `subarray` views instead of copies for packet framing, significantly reducing GC pressure.
- **Synchronous Microtask Batching**: (Postgres) Optimized the streaming engine to process multiple rows in a single synchronous loop per socket read, eliminating microtask latency between rows.

### Industrial-Grade Security Hardening

- **Timing-Safe Zero-Trust**: Replaced standard string comparison with constant-time equality checks for AST signatures to prevent timing attacks.
- **Secure Random Signatures**: Switched to `crypto.getRandomValues()` for signature generation to prevent predictability.
- **DoS & Resource Exhaustion Protection**:
    - Implemented strict `MAX_MESSAGE_SIZE` (16MB) limits at the protocol level.
    - Added LRU-based eviction for Prepared Statement caches (capped at 1000) to prevent memory leaks.
    - Introduced recursion depth limits (8 levels) and dimension validation for nested array decoding to prevent stack overflow attacks.
- **Bound-Checked Protocol Parsers**: Added exhaustive boundary checks to every slice and offset calculation in PostgreSQL and MySQL protocol implementations.

## [1.1.0] - The "Native Supremacy" Release

### Major Features

- **Pureq Native Engines (v2)**: Completely rewritten PostgreSQL and MySQL wire protocol implementations from scratch. Zero external dependencies, pure TypeScript, and fully Edge-compatible (Cloudflare Workers, Bun, Deno).
- **Zero-Trust Execution Mode**: Introduced cryptographic AST-signature validation. When enabled, the driver rejects any raw SQL string not generated by the Pureq Query Builder, physically preventing SQL injection.
- **Protocol-Level Pipelining**: (Postgres) Send multiple queries in a single TCP packet. Reduces network round-trips to exactly one (1RTT), achieving up to 3x performance gains in high-latency environments.
- **Hybrid JIT Decoding**: A world-class row decoding engine that automatically switches between Eager and Lazy (Proxy-based) modes based on row width, resulting in up to 7.8x faster processing for large result sets.
- **Virtual Database (Native Snapshots)**: Record DB traffic and replay it for millisecond-fast CI/CD tests without requiring a real database or Docker.
- **Prepared Statement Caching**: Connection-level caching of statements to skip Parse/Describe phases, reducing DB CPU and bandwidth.
- **Binary-First Streaming**: AsyncIterable cursor mode for processing millions of rows with constant O(1) memory footprint.

### Hardcore Security & Reliability

- **SCRAM-SHA-256 & caching_sha2_password**: Fully native implementations of modern authentication protocols using only standard Web Crypto API.
- **Extended EWKB (PostGIS) Support**: Built-in binary parser for geographical data types.
- **Pluggable Decoders**: Register custom OIDs to handle any specialized database data type.
- **Massive Parameter Protection**: Recursive parameter counting to prevent PostgreSQL 65k limit crashes.
